2. What is Conditional Coverage?

Conditional Coverage is a software testing metric.

It measures whether:

Each Boolean expression (True/False condition)

Inside control structures such as:

if

while

for

has been evaluated to both TRUE and FALSE during execution.

Why Conditional Coverage is Important:

Detects hidden logical bugs

Improves test quality

Helps refactor legacy code safely

Builds confidence in business logic

Reduces production failures

A higher percentage means better logical testing coverage.

3. Step-by-Step Workflow (Python Module)
STEP 1: Select Python Tool

Open TRUSTINN platform

Select Python from language options

Ensure Python tab is active before proceeding

STEP 2: Select Security Tool

In "Tool Configuration"

Choose tool from dropdown

Example: Condition Coverage Fuzzing

Tool must be selected before uploading program

STEP 3: Provide Python Program

User has three options:

Option A: Upload Python File

Click “Upload File”

Click “Browse”

Select .py file

Ensure tool is already selected

Option B: Load Sample Code

Click “Load Sample Code”

System loads predefined Python program for testing

Option C: Write Code in Editor

Click “Write Code”

Code editor opens

Type Python program manually

STEP 4: View Uploaded Program

Click “View File”

System displays full Python source code

Verify correctness before execution

STEP 5: Execute Program

Click “Execute”

Execution begins

Terminal window displays output

Output includes:

Compilation (Syntax Check): SUCCESS

Assertion Summary

Total assertion violations

Total assertion failures

Total assertions added

Conditional Coverage Percentage

Example:
Conditional Coverage: 95%

4. Understanding Output
Line 1:

Compilation (Syntax Check): SUCCESS
→ Python file compiled successfully

Assertion Summary:

Indicates testing analysis has started

Assertion Details:

Shows:

Passed assertions

Failed assertions

Violations

Final Line:

Conditional Coverage: XX%

This percentage indicates:
How thoroughly logical branches were tested.
