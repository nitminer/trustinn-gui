## 📌 Executive Summary

**TrustInn** is an enterprise-grade **Security Analysis & Code Verification Platform** that democratizes advanced software testing tools. We integrate 9 NASA-grade verification tools into a single, intuitive web interface—eliminating complexity, reducing time-to-insight, and making formal verification accessible to developers of all levels.

### The Problem We Solve
- Traditional security tools require **command-line expertise** and months of learning
- Running 9 different tools manually is **time-consuming and error-prone**
- Security analysis results are **hard to visualize and interpret**
- Enterprise tools are **prohibitively expensive** for small teams
- There's **no unified dashboard** for comprehensive security metrics

### Our Solution
A **cloud-based, all-in-one platform** where developers simply:
1. Upload code
2. Select tools
3. Get instant visual results
4. Download comprehensive reports

## 📊 Execution Limits Explained

### What is an "Execution"?
One execution = running ONE tool on ONE code file

**Examples**:
- Uploading `program.c` and running CBMC = **1 execution**
- Uploading `program.c` and running KLEE = **1 execution**
- Uploading `program.c` and running both CBMC + KLEE = **2 executions**
- Uploading 5 files and running KLEE on each = **5 executions**

## 📈 What Each Tool Offers

### 1. CBMC (Condition Satisfiability Analysis)
**What it does**: Determines which code paths are reachable
**Output metrics**:
- ✅ Reachable paths count
- ✅ Unreachable paths count
- ✅ Coverage percentage

**Best for**: Path coverage analysis, proving unreachable code

**Execution time**: 2-30 seconds
**Cost per execution**: Included in all plans

---

### 2. KLEEMA (DSE-based Mutation Analyzer)
**What it does**: Creates mutations of your code and tests them
**Output metrics**:
- ✅ Killed mutants (found by tests)
- ✅ Alive mutants (missed by tests)
- ✅ Mutation score percentage
- ✅ Reached mutants
- ✅ Dead mutants

**Best for**: Evaluating test suite quality, finding weak tests

**Execution time**: 5-120 seconds
**Cost per execution**: Included in all plans

---

### 3. KLEE (Dynamic Symbolic Execution)
**What it does**: Explores all code paths and generates test cases
**Output metrics**:
- ✅ Visited basic blocks
- ✅ Total basic blocks
- ✅ Covered conditions
- ✅ Total conditions
- ✅ Instructions executed
- ✅ Completed paths
- ✅ Generated test cases

**Best for**: Automatic test case generation, comprehensive coverage

**Execution time**: 10-300 seconds
**Cost per execution**: Included in all plans

---

### 4. TX (Pruned Dynamic Symbolic Execution)
**What it does**: KLEE with intelligent pruning for faster execution
**Output metrics**:
- ✅ All KLEE metrics
- ✅ Reduced nodes count
- ✅ Visited blocks total
- ✅ Subsumption checks

**Best for**: Large program analysis, time-sensitive testing

**Execution time**: 5-120 seconds (faster than KLEE)
**Cost per execution**: Included in all plans

---

### 5. gMCov (Coverage Profiler)
**What it does**: Measures MC/DC coverage (Modified Condition/Decision Coverage)
**Output metrics**:
- ✅ MC/DC coverage sequences
- ✅ Coverage percentage
- ✅ Condition coverage details
- ✅ Decision coverage details

**Best for**: Aviation/automotive/safety-critical certification (DO-178C, IEC 61508)

**Execution time**: 5-60 seconds
**Cost per execution**: Included in all plans
